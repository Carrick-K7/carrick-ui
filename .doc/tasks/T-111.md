# T-111: 移动端导航重新设计为底部 Drawer

## 目标
将下拉菜单改为底部 Drawer 设计，参考 iOS/Android 原生交互

## 当前问题
1. 下拉菜单占满屏幕，视觉压抑
2. 滑动时导致页面滚动（事件冒泡未阻止）

## 新设计（底部 Drawer）
- 从**底部向上滑出**，像 iOS Sheet
- 只占用屏幕 **60-70% 高度**
- 顶部有**拖拽指示条**（━━━）
- 可下滑关闭，也可点击遮罩关闭
- 圆角顶部（16px）

## 交互细节
1. 点击 ☰ → Drawer 从底部滑出 + 背景渐暗
2. 点击菜单项 → 跳转 + 自动关闭
3. 点击遮罩 → 关闭
4. 下滑拖拽 → 关闭（可选）
5. 阻止 touchmove 事件冒泡 → 防止页面滚动

## 样式规范
```css
.mobile-drawer {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  height: 70vh;
  max-height: 500px;
  border-radius: 16px 16px 0 0;
  background: var(--miku-card-bg);
  transform: translateY(100%) → translateY(0);
}

.drawer-handle {
  width: 40px;
  height: 4px;
  border-radius: 2px;
  background: var(--miku-text-muted);
  margin: 12px auto;
}
```

## 修改文件
- `carrick-ui/src/components/TopNavbar/TopNavbar.vue`
  - 移除现有 `.mobile-dropdown` 代码
  - 新增底部 Drawer 结构
  - 添加 touch 事件阻止冒泡

## 阻止页面滚动
```javascript
// 打开时
onMounted(() => {
  document.body.style.overflow = 'hidden'
})

// 或阻止 touchmove
@touchmove.prevent="onTouchMove"
```
