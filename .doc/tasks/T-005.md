# T-110: carrick-ui P0 组件精细抽取

## 目标
按 P0 优先级抽取 8 个组件到 carrick-ui，要求精细作业、多项目复用、移动端适配

## 抽取组件（P0优先级）

### 1. CBadge 徽章/状态标签
**用途**: 状态指示、标签展示
**场景**: 任务状态、项目标签、消息通知
**要求**:
- 支持多种颜色（成功、警告、错误、信息、自定义）
- 支持圆点/标签两种形态
- 支持可关闭

### 2. CStatusDot 状态点
**用途**: 在线/离线状态指示
**场景**: 用户状态、服务状态、任务状态
**要求**:
- 支持多种颜色
- 支持脉冲动画
- 支持带文字

### 3. CProgress 进度条
**用途**: 进度展示
**场景**: 项目进度、上传进度、任务完成度
**要求**:
- 线形/环形两种形态
- 支持显示百分比
- 支持动画

### 4. CSwitch 开关
**用途**: 布尔值切换
**场景**: 设置项、功能开关
**要求**:
- 支持不同尺寸
- 支持加载状态
- 支持禁用

### 5. CEmpty 空状态
**用途**: 数据为空时展示
**场景**: 空列表、空页面
**要求**:
- 支持自定义图标
- 支持自定义文字
- 支持操作按钮

### 6. CLoading 加载
**用途**: 加载状态展示
**场景**: 数据加载、提交中
**要求**:
- 支持多种尺寸
- 支持全屏/局部
- 支持自定义文字

### 7. CAvatar 头像
**用途**: 用户头像展示
**场景**: 用户列表、评论、聊天
**要求**:
- 支持图片/文字两种模式
- 支持不同尺寸
- 支持状态指示点

### 8. CTabs 标签页
**用途**: 内容分组切换
**场景**: 详情页、设置页
**要求**:
- 支持横向/纵向
- 支持卡片/线条样式
- 支持禁用某一项

## 质量要求

### 1. 多项目复用
- Props 设计通用，不依赖业务数据
- 事件命名规范（@change, @click等）
- 支持插槽自定义内容

### 2. 样式统一
- 使用 CSS 变量（--miku-primary, --miku-text等）
- 颜色、圆角、阴影遵循 Design System
- 暗色模式自动适配

### 3. 移动端适配
- 触摸目标 >= 44px
- 响应式布局
- 移动端手势支持

### 4. 文档完整
- 每个组件必须有 README.md
- Props/Events/Slots 说明
- 使用示例

## 开发流程
每个组件严格按以下流程：
1. **文档先行**: README.md（Props设计、使用示例）
2. **开发实现**: Vue单文件组件
3. **暗色适配**: 确保暗色模式正常
4. **移动端测试**: 触摸、响应式
5. **QA检查**: 无TODO, build通过
6. **更新索引**: index.js 导出
